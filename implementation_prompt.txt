PHASE 4 — GET-REDIRECT ENDPOINT FOR WHATSAPP CLICK TRACKING (10 min)
--------------------------------------------------------------------------------

GOAL: When a lead taps the WhatsApp link, our backend logs the click and
302-redirects to the real Rupeezy signup URL.

The current /calls/track-click endpoint is a POST — phones can't hit it.
We need a public GET endpoint that takes the lead_id in the path.

FILE: app/main.py — add this route:

  from fastapi.responses import RedirectResponse
  from sqlalchemy import select
  from sqlalchemy.ext.asyncio import AsyncSession
  from datetime import datetime, timezone

  from app.database import get_db
  from app.models import Call, WhatsAppMessage

  @app.get("/r/{lead_id}")
  async def public_redirect(lead_id: str, db: AsyncSession = Depends(get_db)):
      """Public click-tracking redirect for WhatsApp links."""
      result = await db.execute(
          select(WhatsAppMessage)
          .join(Call)
          .where(Call.lead_id == lead_id)
          .order_by(WhatsAppMessage.sent_at.desc())
      )
      wa = result.scalars().first()
      if wa and not wa.clicked_at:
          wa.clicked_at = datetime.now(timezone.utc)
          await db.commit()
      target = f"{settings.RUPEEZY_SIGNUP_BASE_URL}{lead_id}"
      return RedirectResponse(url=target, status_code=302)

PHASE 5 — ENHANCE SYSTEM PROMPT (15 min)
--------------------------------------------------------------------------------

GOAL: The agent should track emotional cues, adapt tone, and never
sound scripted.

FILE: app/prompts/system_prompt.py — INSERT this new section into
SALES_SYSTEM_PROMPT before "## Output format":

  ## Mood & Conversation Steering
  Read the lead's mood every turn and adapt:

  - HIGH ENERGY (excited, asking many questions, "tell me more") →
    Push toward CTA quickly. Ask one qualifying question, then offer
    sign-up or RM call.

  - HESITANT (long pauses, "hmm", "let me think", "maybe") →
    Soften. Offer "main aapko WhatsApp pe details bhej deta hoon, aap
    apne pace pe dekh lo." Don't push.

  - FRUSTRATED ("I don't have time", abrupt tone, sighs) →
    Respect it. 30-second pitch max, then offer WhatsApp and exit:
    "Bilkul samajhta hoon, aap busy hain. Main 30 second mein WhatsApp
    bhej deta hoon, badme dekh lijiye."

  - SKEPTICAL ("really?", "are you sure?", "kaise trust karein?") →
    Lead with proof: SEBI registration, ₹500 Cr+ AUM, 5-star Play Store
    rating, 3-year track record. Offer to send fact sheet on WhatsApp.

  - ENGAGED (asking specifics like "kitna milega", "kab payout aata hai") →
    You're winning. Answer concretely (100% brokerage, daily, T+1).
    Then qualify network size and push for sign-up.

  Lead says "not interested" twice → exit gracefully:
    "Bilkul, no problem. Main WhatsApp pe info chhod deta hoon, future
    mein interest ho toh batayiega." Then end the call politely.

  HARD RULES:
  - Never repeat a benefit you've already stated.
  - Never pressure. Never use guilt or scarcity.
  - Never argue about the lead's current broker — acknowledge first.
  - If you don't know something (specific tax rule, exact onboarding date),
    say "main aapko exact info WhatsApp kar deta hoon" instead of guessing.

  ## Language — Match Strictly
  - Detect language from the lead's FIRST response, not just from the
    declared preference. Override preference if needed.
  - Match the lead's exact register: Hinglish stays Hinglish (do not switch
    to pure Hindi or pure English mid-call). Tamil-English mix → match it.
  - Match formality: "aap" if they use "aap"; "tum/tu" only if they do.
  - SHORT answers. 1–2 sentences per turn. This is a phone call, not an essay.

After updating in code, COPY the rendered full prompt (with placeholders left
intact) and PASTE it into the ElevenLabs dashboard system prompt. ElevenLabs
holds the live prompt — our system_prompt.py is just the source of truth.

VERIFICATION
  Test in ElevenLabs UI with three roleplay flows:
    (a) "Wow, tell me everything!"  → should drive to CTA quickly
    (b) "Hmm, let me think about it" → should offer WhatsApp, soften
    (c) "I'm busy, I don't have time" → should exit gracefully in 30s

        """Send a freeform text message. Only works inside the 24h service window."""
        url = f"{GRAPH_BASE}/{settings.WHATSAPP_PHONE_NUMBER_ID}/messages"

            "to": to,
            "type": "text",
            "text": {"body": body, "preview_url": True},
        }

            r = c.post(url, headers=headers, json=payload)
            r.raise_for_status()

            "link": link,
            "language": language,
            "twilio_sid": wa_id,    # column is named twilio_sid; reuse it for wa message id
        }

  FILE: app/routers/whatsapp.py — create with:

    """Meta WhatsApp Cloud API webhook for delivery/read/reply status."""

    from datetime import datetime, timezone

    from fastapi import APIRouter, Depends, Query, Request
    from sqlalchemy import select
    from sqlalchemy.ext.asyncio import AsyncSession

    from app.config import settings
    from app.database import get_db
    from app.models import WhatsAppMessage

    router = APIRouter(prefix="/whatsapp", tags=["whatsapp"])


    @router.get("/webhook")
    async def verify(
        mode: str = Query(..., alias="hub.mode"),
        token: str = Query(..., alias="hub.verify_token"),
        challenge: str = Query(..., alias="hub.challenge"),
    ):
        """Meta calls this once to verify webhook ownership."""
        if mode == "subscribe" and token == settings.META_VERIFY_TOKEN:
            return int(challenge)
        return {"error": "verify token mismatch"}


    @router.post("/webhook")
    async def receive(req: Request, db: AsyncSession = Depends(get_db)):
        body = await req.json()
        for entry in body.get("entry", []):
            for change in entry.get("changes", []):
                value = change.get("value", {})

                # Status updates: sent / delivered / read / failed
                for status in value.get("statuses", []):
                    wa_id = status.get("id")
                    state = status.get("status")
                    ts = datetime.now(timezone.utc)
                    res = await db.execute(
                        select(WhatsAppMessage).where(WhatsAppMessage.twilio_sid == wa_id)
                    )
                    wa = res.scalar_one_or_none()
                    if not wa:
                        continue
                    if state == "delivered" and not wa.delivered_at:
                        wa.delivered_at = ts
                    elif state == "read" and not wa.read_at:
                        wa.read_at = ts
                    await db.commit()

                # Inbound messages: lead replied
                for msg in value.get("messages", []):
                    from_phone = msg.get("from", "")
                    res = await db.execute(
                        select(WhatsAppMessage)
                        .join(WhatsAppMessage.call)
                        .order_by(WhatsAppMessage.sent_at.desc())
                    )
                    # Best-effort: match by phone via the related lead
                    for wa in res.scalars():
                        if wa.call.lead.phone.lstrip("+") == from_phone and not wa.replied_at:
                            wa.replied_at = datetime.now(timezone.utc)
                            await db.commit()
                            break

        return {"ok": True}

PHASE 8 — TEMPLATE OR SERVICE-WINDOW STRATEGY (10–60 min)
--------------------------------------------------------------------------------

WhatsApp Cloud API rule (important):
  - Business-INITIATED messages to a contact who hasn't messaged you in
    the past 24h MUST use a pre-approved template.
  - Once the contact messages you, you have a 24h service window to send
    freeform messages. After 24h of silence, you need template again.

For the hackathon demo with your test phone, you have two paths:

PATH A — DEMO PATH (zero waiting)

  Before each demo session, use your test phone (+918102652783) to send any
  message ("Hi") to the Cloud API number. This opens the 24h window. After
  that, send_warm_followup() will hit the freeform path successfully.

  Pros:  works in 30 seconds, no Meta approval wait
  Cons:  every recipient must message us first; not realistic for production

PATH B — PRODUCTION PATH (template approval)

  1. Meta Business Manager → WhatsApp Manager → Message Templates →
     "Create Template".
  2. Name:       rupeezy_ap_signup
     Category:   MARKETING
     Language:   English (start; add Hindi/Hinglish later)
  3. Body:
       Hi {{1}}, great chatting about the Rupeezy AP Program!
       Zero joining fee · 100% brokerage share · Daily payouts.
       Tap below to sign up — takes 2 minutes.
  4. Buttons → "Call to action" → "Visit website":
       Type:                   Dynamic
       URL pattern:            https://NGROK_URL/r/{{1}}
       Button text:            Sign up
  5. Submit. Approval is usually 1–24 hours for Marketing templates.

  Once approved, the fallback in send_warm_followup() (Phase 6 code) hits
  this template automatically when the freeform path fails.

  WARNING about ngrok URLs in templates:
    Meta locks template URLs at approval time. If your ngrok URL changes,
    the template breaks. For demo, use Path A. For production, deploy to
    Render/Railway/Vercel and use a stable URL in the template.

PICK PATH A FOR THE HACKATHON. Note Path B as production roadmap.

APPENDIX C — TROUBLESHOOTING
================================================================================

SYMPTOM: CSV uploads but no call comes
  CHECK: ELEVENLABS_PHONE_NUMBER_ID is set in .env
  CHECK: docker compose logs api → look for the outbound HTTP call
  CHECK: ElevenLabs dashboard → Conversation history → did a call attempt log?

SYMPTOM: Call connects but agent doesn't say my name
  CHECK: dynamic_variables payload field name (variables vs dynamic_variables)
  CHECK: ElevenLabs system prompt uses {{lead_name}} not {lead_name}
  CHECK: First message field uses {{lead_name}}

SYMPTOM: Call ends but no transcript saved
  CHECK: NGROK_URL/voice/post-call is reachable from the internet
  CHECK: ElevenLabs agent → Webhooks → Post-call → URL matches current ngrok
  CHECK: docker compose logs api → 200 on /voice/post-call?

SYMPTOM: Worker doesn't process the call
  CHECK: docker compose logs worker — is it picking up jobs?
  CHECK: GEMINI_API_KEY is valid (test with curl to Gemini API)
  CHECK: scorer.py JSON parsing — Gemini sometimes wraps in ```json fences

SYMPTOM: WhatsApp send returns 401
  CHECK: WHATSAPP_ACCESS_TOKEN is current (24h temp tokens expire fast)
  FIX:   Generate a permanent System User token in Meta Business Settings

SYMPTOM: WhatsApp send returns 400 code 131047
  CAUSE: 24h window closed and no template OR template not approved
  FIX (demo):   Send "Hi" from test phone to Cloud API number, retry
  FIX (prod):   Wait for template approval, ensure name + lang match .env

SYMPTOM: Click on WhatsApp link doesn't track
  CHECK: WhatsApp link in message points to NGROK_URL/r/{lead_id}, not direct
  CHECK: docker compose logs api → GET /r/<id> appears?
  CHECK: clicked_at update SQL ran (look for commit in logs)

SYMPTOM: ngrok URL changed mid-demo
  FIX: docker compose down && update .env NGROK_URL && docker compose up -d
       Then update ElevenLabs post-call webhook URL + Meta webhook URL
